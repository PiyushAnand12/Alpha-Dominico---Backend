What does this screener actually do?

Plain English explanation before you start

Every evening after the US stock market closes, this program:

📥

Downloads all US stocks

Fetches 1 year of price history for ~6,000 stocks from Yahoo Finance (free).

📐

Checks the Trend Template

Tests all 8 technical criteria (moving averages, 52-week position, RS rank). Think of this as the "is the stock in the right shape?" check.

💰

Checks the Fundamentals

For every stock that passes the technical check, looks up earnings growth, revenue growth, and margin trend.

📨

Sends you a report

Emails you a clean HTML report and a Telegram message listing every stock that currently meets all criteria.

⚠️
Important: This tool tells you which stocks currently meet the criteria. It does NOT tell you to buy anything. It does not predict the future. Always do your own research before making any investment decision.

What the output looks like:

NVDA ✅ FULL RS: 94 EPS: +85% Rev: +42% Conf: 88 · Technology
AXON ✅ FULL RS: 91 EPS: +62% ⚡ Rev: +31% Conf: 79 · Industrials
META 📋 TECH ONLY RS: 82 EPS data missing

⚡ = EPS growth accelerating. "FULL" = passed both technical + fundamental layers. "TECH ONLY" = technical passed but fundamental did not.

2

Create your screener folder

Put all the files in one place so Python can find them

Create a folder on your computer called sepa_screener. You can do this the normal way (right-click → New Folder) or via terminal. Then copy the three files from this package into it:

📁 sepa_screener/ ├── screener.py ← the main program ├── requirements.txt ← list of packages to install └── .env ← your private settings (you create this next)
✅
The .env file does not exist yet — you'll create it in Step 4. For now just copy screener.py and requirements.txt into the folder.

Now open your terminal inside that folder. The easiest ways:

  • Windows: Hold Shift, right-click inside the folder → "Open PowerShell window here" (or Command Prompt)
  • Mac: In Finder, right-click the folder → Services → "New Terminal at Folder"
  • Any OS: Open terminal, then type cd followed by the path to your folder:
cd C:\Users\YourName\sepa_screener # Windows example cd /Users/YourName/sepa_screener # Mac/Linux example
3

Install the required packages

These are free libraries that add stock-fetching and data-processing abilities to Python

Think of packages like apps on your phone — they add extra features. This screener needs a few:

  • yfinance — downloads stock price and financial data from Yahoo Finance (free)
  • pandas — works with tables of data (like Excel but in code)
  • numpy — fast math operations (moving averages, etc.)
  • requests — fetches data from the internet
  • python-dotenv — reads your settings from the .env file safely

With your terminal open inside the sepa_screener folder, run this single command:

Install everything at once
pip install -r requirements.txt
💡
On Mac/Linux, you may need to use pip3 instead of pip. If you get a "command not found" error, try python -m pip install -r requirements.txt

You'll see a bunch of text scrolling by — that's normal. It's downloading and installing. When it finishes and you see your terminal prompt again, you're done. ✅

✅
Optional but recommended — use a virtual environment: This keeps the screener's packages separate from other Python projects. Run python -m venv venv, then activate it with venv\Scripts\activate (Windows) or source venv/bin/activate (Mac/Linux), then run the pip install command above.
4

Set up your credentials

The screener needs a way to email you and (optionally) message you on Telegram

Create a file called exactly .env (note the dot at the start) inside your sepa_screener folder. Open it with any text editor (Notepad on Windows, TextEdit on Mac). Copy and paste this template:

Contents of your .env file
EMAIL_SENDER=your_gmail@gmail.com EMAIL_PASSWORD=xxxx xxxx xxxx xxxx EMAIL_TO=your_gmail@gmail.com TELEGRAM_BOT_TOKEN= TELEGRAM_CHAT_ID=

📧 Setting up Gmail (required for email reports)

You need a Gmail App Password — this is a special 16-character code that lets the screener send emails on your behalf, without using your real Gmail password. It only works if you have 2-Step Verification turned on in Gmail.

1
Go to your Google Account settings Visit myaccount.google.com and click Security in the left sidebar
2
Turn on 2-Step Verification if it's off Under "How you sign in to Google", click 2-Step Verification and follow the steps. You need this on before you can create an App Password.
3
Create an App Password Go back to Security → scroll down → click "App passwords". Type "SEPA Screener" in the name box → click Create. Google will show you a 16-character code like abcd efgh ijkl mnop. Copy it immediately — you can only see it once.
4
Paste it into your .env file Replace xxxx xxxx xxxx xxxx with your App Password (spaces are fine). Also replace your_gmail@gmail.com with your actual Gmail address in both the SENDER and TO fields.

📱 Setting up Telegram (optional — for instant messages)

If you want a quick Telegram message when the screener runs, follow these steps. If you just want email, leave TELEGRAM_BOT_TOKEN and TELEGRAM_CHAT_ID blank.

1
Create a Telegram Bot Open Telegram and search for @BotFather. Send it the message /newbot. It will ask for a name (e.g. "My Screener") and a username (must end in "bot", e.g. "myscreener_bot"). It will then give you a token that looks like 123456789:ABCdef... — copy it.
2
Get your Chat ID Search for @userinfobot in Telegram. Send it any message (like "hi"). It will reply with your ID number, like 123456789. Copy that number.
3
Paste both into your .env file Put the bot token after TELEGRAM_BOT_TOKEN= and your chat ID after TELEGRAM_CHAT_ID=. Also send a message to your own bot in Telegram first (just open the chat and press Start) so it can message you.
🔒
Keep your .env file private. Never upload it to GitHub, share it in a chat, or show it to anyone. It contains your Gmail App Password. If you accidentally share it, immediately go to Google Account → Security → App Passwords and delete that app password, then create a new one.

Common errors and fixes

!
"ModuleNotFoundError: No module named 'yfinance'" You didn't install the packages yet. Run pip install -r requirements.txt first.
!
"SMTPAuthenticationError" when sending email Your Gmail App Password is wrong or you used your regular password. Double-check Step 4.
!
"python is not recognized as a command" (Windows) Python wasn't added to PATH. Re-install Python and make sure to check "Add Python to PATH".
!
Screener runs but sends 0 qualifiers That's valid — it means no stocks meet all criteria on that day. The market breadth could be low. Try running on a different day, or check the "Close Calls" section in the email/HTML report.
!
Very slow / hangs on "Downloading price history" Normal — downloading 2 years of data for 6,000 stocks takes time. A full run takes 30–60 minutes depending on your internet speed. Don't close the terminal window.

Run the full screener

Scan all ~6,000 US stocks — typically takes 30–60 minutes

With MAX_TICKERS removed from your .env (or set to 0), run the same command as before:

python screener.py # Windows python3 screener.py # Mac / Linux

You can minimize the terminal window — it will keep running in the background. Don't turn off your computer until it finishes. When complete:

  • An HTML report is saved in your folder — open it in any browser
  • An email is sent to your Gmail
  • A Telegram message is sent (if configured)
  • A screener.log file records everything that happened
💡
Best time to run: After 5:00 PM Eastern Time (US markets close at 4:00 PM ET, and data takes ~30 min to fully populate). If you run at 6 PM or later, you'll have complete end-of-day data for all stocks.

Automate it — run every weekday automatically

Set it and forget it. The screener runs itself every evening.

You can make the screener run automatically every weekday evening using your operating system's built-in scheduler. No extra software needed.

🪟 Windows Task Scheduler
🍎 Mac cron
🐧 Linux cron
  1. Press Windows key, search for Task Scheduler, open it
  2. In the right panel, click Create Basic Task
  3. Name: SEPA Screener → click Next
  4. Trigger: select Daily → click Next
  5. Start time: 6:00 PM, recur every 1 day → click Next
  6. Action: select Start a program → click Next
  7. Program/script: Browse to your Python executable. Usually: C:\Users\YourName\AppData\Local\Programs\Python\Python312\python.exe
  8. Add arguments: screener.py
  9. Start in: your screener folder path, e.g. C:\Users\YourName\sepa_screener
  10. Click Next → Finish

To run only on weekdays, after creating the task: right-click it → Properties → Triggers tab → Edit → Advanced settings → check "Stop task if it runs longer than" → also go to Conditions tab and check settings. Alternatively, add a check inside the script (already handled — the screener runs any day but markets are only open Mon–Fri, so weekday results will differ from weekend runs).

✅
To test your scheduled task, right-click it in Task Scheduler → Run. The screener should start.

Open Terminal and edit your cron schedule:

crontab -e

This opens a text editor. Press i to enter insert mode, then add this line (replacing the path with your actual folder path):

0 18 * * 1-5 cd /Users/YourName/sepa_screener && python3 screener.py >> screener.log 2>&1

This means: at 6:00 PM, Monday through Friday, go into the screener folder and run the screener. Press Esc, then type :wq and press Enter to save.

⚠️
Your Mac must be awake at 6 PM for the cron job to run. If your Mac sleeps, go to System Settings → Battery → uncheck "Enable Power Nap" and consider keeping the lid open or plugged in at that time.
crontab -e

Add this line (replace the path):

If running on a server (VPS), it will run even while your own computer is off — ideal for always-on operation.

✅
Running on a $5/month VPS (like DigitalOcean, Vultr, or Oracle Cloud Free Tier) is the most reliable approach. The server is always on, always running. Oracle Cloud has a permanently free VPS tier.

Understanding the output

What each column means in plain English

Column What it means What's good?
ConfidenceA 0–100 score based on how strongly the stock meets the optional criteria. Higher = more criteria met = stronger candidate on paper.70+ is strong
RS RankRelative Strength rank vs. all other US stocks. 90 means the stock has outperformed 90% of all other US stocks over the past 12 months.80+ is strong; 90+ is elite
EPS Growth QHow much the company's earnings grew vs. the same quarter last year. ⚡ means the growth rate is accelerating (getting bigger each quarter).20%+ required; 50%+ is excellent
Rev Growth QHow much the company's sales revenue grew vs. the same quarter last year. Revenue growth confirms that earnings aren't just from cost-cutting.15%+ required; 30%+ is strong
FULL vs TECH ONLYFULL = passed both technical AND fundamental criteria. TECH ONLY = passed the 8 Trend Template criteria but fundamental data was missing or did not meet thresholds.Focus on FULL qualifiers first
💡
What to do with the list: The report is a starting point, not a shopping list. When you see a stock you don't recognise, look it up. Read what the company does, look at its chart, understand why it's growing. Never buy something just because it appeared in a screen.

Customise the thresholds (optional)

Adjust how strict the screening is by editing your .env file

The defaults are based on the methodology as described, but you may want to adjust them. All of these go in your .env file:

Setting Default What it does Raise it to...
EPS_GROWTH_MIN20Min quarterly EPS growth %25–50 for stricter filtering, fewer results
REV_GROWTH_MIN15Min quarterly revenue growth %20–30 for higher-growth-only results
RS_RANK_MIN70Min RS percentile rank80 or 85 to focus on top leaders only
PCT_FROM_52WH25Must be within X% of 52-week high15 for stocks very close to breaking out
MIN_AVG_VOL100000Minimum daily volume (shares)500000 to focus on more liquid, larger stocks
⚠️
More results ≠ better results. Lowering thresholds gives you more stocks but dilutes quality. Start with the defaults, run the screener for a few weeks, and adjust based on your own judgment after seeing real results.
